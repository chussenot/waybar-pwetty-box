# SUT Analysis — waybar-pwetty-box

Synthesized from a 12-focus ensemble analysis (architecture, state, concurrency,
safety claims, liveness claims, bug history, test strategy, failure modes,
dependencies, product context, unproven assumptions, wildcard). Provenance tags:
`[arch] [state] [conc] [safety] [live] [bugs] [tests] [fail] [deps] [product]
[assume] [wild]`.

## Summary

waybar-pwetty-box is a Rust cdylib (`libpwetty_box.so`) loaded **in-process**
by waybar (GTK3 Wayland status bar) through waybar's CFFI module ABI. It
renders animated "tiles" — GL shader backgrounds plus Pango/Cairo text/icon
flows — showing live Claude Code session state per niri desktop. Data comes
from an external Go backend (`claude-status` in ~/agentic-db) over a
newline-delimited-JSON pipe. The plugin holds no durable state; it is a pure
render sink.

The dominant risk shape is **shared fate with the host**: any abort or panic in
the plugin kills the whole waybar process (every bar on the desktop). The
second risk shape is **silent staleness**: every failure in the data chain
renders as calm, plausible-looking normality, masking the one state the
product exists to surface (`prompt` — "Claude is waiting for you"). Both known
crash bugs and most latent hazards live in lifecycle/teardown timing and the
cross-process stream seam — exactly Antithesis territory.

## 1. Architecture and Data Flow `[arch]`

```
Claude Code hooks ──> claude-status daemon (Go, actor model)      [~/agentic-db]
                        │  SQLite session DB (13ms poll + 13ms debounce)
                        │  niri event-stream subscription
                        │  1s tick → tiles.json (atomic tmp+rename, byte-dedupe, 250ms throttle)
                        ▼
        claude-status tile-watch <idx>   (one long-lived child process per tile)
                        │  polls tiles.json every 75ms, prints 1 JSON line on change
                        │  (single Write per line; initial line immediately at start)
                        ▼ pipe, newline-delimited JSON
waybar (GTK3, C++) ── dlopen(libpwetty_box.so), never dlclose ── wbcffi_init
                        │  detached reader thread → ContentStore (Arc<Mutex> + AtomicBools)
                        ▼
        GTK main thread: 150ms dirty poll + frame-clock tick → queue_draw
          → draw signal → own surfaceless-Mesa EGL make_current (per frame, GL tiles only)
          → shader render + glReadPixels → Cairo composite
          → minijinja template → roxmltree markup route → Pango text flow → hover ring
```

- **FFI surface** (validated against the built .so, waybar-cffi 0.1.1 crate
  source, and waybar upstream `src/modules/cffi.cpp`): `wbcffi_version` (=1),
  `wbcffi_init`, `wbcffi_deinit`, `wbcffi_update`, `wbcffi_refresh`,
  `wbcffi_doaction`, generated by `waybar_module!(PwettyBox)` (src/lib.rs:1665).
  The plugin implements only `init`; update/refresh/doaction are no-op
  defaults — all repaint scheduling bypasses waybar's own update machinery.
  Waybar treats a null return from `wbcffi_init` as a per-module load failure
  (logged, module skipped); everything else that goes wrong in Rust is fatal
  to the host (§8).
- **Waybar never dlcloses** the .so (verified in cffi.cpp at 0.14.0/0.15.0/
  master, and `dlclose` is absent from /usr/bin/waybar's imports `[wild]`).
  Code stays mapped across reloads; leaks accumulate as instances, not
  use-after-unload.
- **Config flow**: waybar passes each config value as a C string; waybar-cffi
  re-parses each as JSON, falling back to string (crate config.rs:39-53). The
  plugin deep-merges a bundled/`tile_file` preset under the raw config
  (src/config.rs:79-124) and deserializes; a hard error collapses the whole
  config to `Config::default()` with only a stderr log (src/config.rs:139-142).
- **Two content modes** (src/content.rs:234-296): poll (`exec` + `interval`,
  `sh -c`, whole stdout as one value) and stream (`stream: true`, spawn once,
  one value per stdout line, EOF → keep last content + respawn after fixed 1s
  backoff, forever). The claude preset ships `stream: true`.
- **GL handoff**: the plugin creates its own surfaceless-Mesa EGL/GLES3 context
  (`EGL_PLATFORM_SURFACELESS_MESA`, src/offscreen.rs:14-66) — no Wayland
  connection, no DRM master, no seat. GL function pointers are resolved from
  the host's `libepoxy.so.0` (src/gl.rs:15-36). Pure-Cairo frames skip
  `make_current` entirely via a substring gate (`needs_gl`, src/lib.rs:255-260).
  The plugin leaves its context current after draws and never restores GDK's
  (§11, F13).

## 2. State Management `[state]`

- Per module instance: `Rc<Shared>{ RefCell<Option<Engine>>, Config }` shared
  with GTK closures (src/lib.rs:84-93); `Engine` owns the EGL context, femtovg
  canvas, shader caches, frame counter (src/lib.rs:41-52). Cross-thread state
  is exactly `ContentStore = Arc<Mutex<TileContent>> + AtomicBool dirty +
  AtomicBool animating` (src/content.rs:36-48). **No durable state in the
  plugin** — a crash loses only animation phase.
- **Engine field order is the p9c crash**: `gl: OffscreenGl` is declared before
  `renderer: Renderer`; Rust drops in declaration order, so on any Engine drop
  the EGL context is unbound/destroyed *first*, then the femtovg canvas issues
  GL deletes with no context current → epoxy assert → SIGABRT of waybar
  (src/lib.rs:41-52, src/offscreen.rs:75-80). No `Drop` guard exists at
  f87ec19; the fix (`ManuallyDrop` + make-current-before-drop, commit 30100f9)
  lives only on unmerged branch `worktree-fix-gl-teardown-crash`.
- **Caches, three strategies, two pathological** `[state][assume]`:
  - `INK_CACHE` (src/lib.rs:591-610): thread-local, keyed by full run-markup
    text, never evicted — grows monotonically with changing titles/ago labels;
    survives module reloads.
  - `ICON_CACHE` (src/lib.rs:1195-1203, 1300-1315): caches **failures
    permanently** per (path, px, tint); an icon file that appears later (e.g.
    the backend's async PNG→SVG shim) stays missing until waybar restarts. No
    mtime in the key, no log.
  - Background shader: mtime-watched, but a *failing* file is re-read,
    re-compiled, and re-logged **every frame** (guard requires
    `shader.is_some()`, src/lib.rs:57-79); each recompile leaks the old
    `ShaderPass` GL objects (no Drop impl).
- **Backend state**: SQLite DB + `tiles.json` cache, written atomically
  (tmp+rename, byte-dedupe, tile.go:431-437) — torn reads ruled out. But the
  cache has no TTL/freshness marker, is never removed on daemon shutdown, and
  `tile-watch` trusts it unconditionally (§8, staleness).

## 3. Concurrency Model `[conc]`

- **Thread inventory**: GTK main thread runs all FFI entrypoints (validated
  against waybar upstream: update/refresh/doaction all dispatch to the main
  loop), all GTK callbacks, and all GL. One detached reader thread per module
  instance (poll or stream). Waybar's signal thread never calls into the
  plugin. Children: one `sh -c <exec>` per module (long-lived `tile-watch` in
  stream mode).
- The waybar-cffi `Box::from_raw → call → into_raw` dance in
  update/refresh/doaction would be UB under concurrent calls; sound in
  practice because all are main-thread-only.
- **ContentStore protocol**: single mutex, writers = reader thread, readers =
  main thread. Two benign one-frame windows: `animating` is stored before the
  content and `dirty` after (src/content.rs:84-92); and the draw callback
  reads `uniforms()` and `markup()` under **two separate lock acquisitions**
  (src/lib.rs:234-247) — a `set()` in between renders one frame with uniforms
  from generation N and markup from N+1. Self-heals next dirty poll.
- Mutex poisoning is silently swallowed (`if let Ok` / `unwrap_or_default`,
  src/content.rs:88, 100-115): a poisoned lock would permanently, silently
  blank the tile. Practically unreachable; ideal `Unreachable` instrumentation
  point.
- RefCell/Rc discipline on the main thread is sound: `Shared.engine` is
  mutably borrowed exactly once (draw), nothing re-enters. Hover timers are
  bounded and self-terminating.
- **Multi-instance reality**: the live deployment is 10 instances in one
  process = 10 EGL contexts + 10 threads + 10 children, all drawing on one
  main thread with per-frame context switching.

## 5. Bug History and Density `[bugs]`

**Confirmed defects (primary evidence, not report headlines):**

- **p9c — SIGABRT on output removal: CONFIRMED at f87ec19.** The agent
  extracted the actual core dump (`/var/crash/_usr_bin_waybar.1000.crash`,
  2026-07-21) and ran gdb: `__assert_fail "Couldn't find current GLX or EGL
  context"` (epoxy) ← `glow delete_vertex_array` ← `femtovg OpenGl::drop` ←
  `Rc::drop_slow` ← `PwettyBox::init::{{closure}}` ← `g_signal_handlers_destroy`
  ← `g_object_run_dispose`. Code legs both confirmed (Engine field order; for
  shader-less content tiles `needs_gl` is false so *nothing* is ever current
  at dispose). Fix unmerged (branch `worktree-fix-gl-teardown-crash`, commit
  30100f9). Fix-completeness checked: femtovg's is the only GL-calling
  destructor; ShaderPass has no Drop, so no residual abort path in that class.
- **q9y — SIGUSR2 reload crash: UNVALIDATED attribution.** The Jul-19 crash
  file was overwritten by the Jul-21 crash (single apport path) — evidence
  destroyed. Mechanism is plausible (the confirmed backtrace's upper frames
  are generic widget-dispose, which reload also triggers; "add worked /
  reconfigure crashed" fits), but not independently confirmed.
- **Reload resource leak: CONFIRMED by code, unfiled.** Detached reader
  threads ("waybar modules are process-lifetime" — false premise,
  src/content.rs:207, 259), perpetually-respawning `sh -c` children, and the
  150ms `timeout_add_local` (always `Continue`, strong widget ref,
  src/lib.rs:366-374) all survive `wbcffi_deinit`. N reloads ⇒ N producer
  chains + N timers per tile. **Currently masked because reload crashes
  first; merging the p9c fix promotes this to the active reload bug.**
- **dsl — 60Hz redraw/heat: fix confirmed in place**, but the area is the
  highest-churn regression cluster: the fix regressed twice within a day
  (4269a03 blink stall — quote-style substring bug; dd610c4 blink crawl —
  per-frame ink rasterize). Three gating blind spots remain (§11).

**Density map:** GL context lifecycle/state (highest severity: bpe, 5d80c5b,
p9c/q9y) → animation gating (highest churn) → Pango-metrics-vs-bitmap-font ink
(most filed bugs: 4up, e0r, 1ml + re-fixes) → color tuning (trivial).

**Suspiciously quiet:** the exec/stream consumer (zero filed bugs at the trust
boundary; only happy-path tests — undertested, not proven), preset deep-merge,
multi-instance operation, the unimplemented ABI surface.

**Backend hotspot** `[bugs][deps]`: agentic-db's state derivation has six fixes
(reaper liveness, idle-nudge strands "?", waiting→"?" false positives,
nondeterministic window pick, stale-`busy` inversion, cwd resolution) — the
session-state inference parses Claude's **undocumented** session-file format.
The tile trusts these state values without validation.

- **The seam nothing tests**: real `tile-watch` stdout → plugin stream reader.

## 7. Failure and Degradation Modes `[fail]`

**Panic/abort sites reachable in-host (exhaustive):**

1. `src/gl.rs:22,25` — `.expect("libepoxy.so.0 should be loadable")`: init
   path AND draw path (via ShaderPass::compile). Missing epoxy = host abort.
   Missing symbols resolve to null and segfault on call (gl.rs:33).
2. `src/shader.rs:277,300` — `create_texture().unwrap()` /
   `create_framebuffer().unwrap()` in the per-frame draw path: GL failure
   (context loss, GPU reset, ID exhaustion) = panic in a GTK signal handler =
   abort. No `glGetError`/`glCheckFramebufferStatus` anywhere in the codebase.
3. Engine drop without current context (p9c) — the confirmed abort.
4. Dependency: waybar-cffi's `wbcffi_doaction` does `.expect()` on non-UTF-8
   action names.

waybar-cffi has **no catch_unwind**; every panic crosses `extern "C"` or a
glib trampoline → process abort. All other unwrap/expect sites are
test/CLI-only.

**Degraded states (the messy middle):**

- Stream: EOF → keep last content + 1s respawn (stale-by-design, no staleness
  indicator). Invalid UTF-8 line → reader breaks, then **blocks in
  `child.wait()` until the child's next write** (EPIPE) — on a quiet desktop
  the tile freezes indefinitely at stale content. Partial final line (kill
  mid-write) → red inline "template error" flash until respawn (~1s).
  No line-length cap → newline-less producer grows host memory unboundedly.
- Poll: `Command::output()` with **no timeout** — hung command freezes that
  tile forever, silently. Failure → `""` → template error card. `interval: 0`
  one-shot racing ahead of the backend at login = wrong forever. Pointing a
  never-exiting streaming producer at poll mode (default `stream: false`)
  buffers all output forever: unbounded memory + permanently blank tile.
- Silent spawn failure: `claude-status` not on waybar's PATH → `sh` exits 127,
  stderr nulled, respawn every 1s forever (~86k forks/day), tile blank, zero
  diagnostics.
- Config: hard deserialize error → **demo tile animating at 60fps** (a heat
  regression via config typo). Unknown preset name → raw config → tile
  renders the raw JSON object as text.
- GL: engine init failure or per-frame `make_current` failure blanks
  **everything including pure-Cairo text** (whole draw body gated on engine,
  src/lib.rs:239, 260); no retry, no context-loss recovery — blank until
  waybar restarts. Shader compile error: ShaderCache caches Err permanently
  (bundled) vs per-frame retry storm (file-based) — inconsistent policies.
- Backend: daemon exits when the niri event stream closes (daemon.go:217-221),
  **no restart unit ships**; cache freezes; tile-watch emits nothing more
  ("no change" and "chain dead" are indistinguishable). Cache read error →
  `emptyPayload` (idle level-6) — a live `prompt` visually replaced by a
  plausible "long idle". Transient cache-miss during daemon restart actively
  *pushes* placeholder over good state.
- Design posture: no timeouts anywhere, no health signaling, "log to stderr,
  keep last content or go blank, never retry what failed once."

## 9. Product Context and Failure Severity `[product]`

Personal tool; live deployment = 10 `cffi/pwetty#N` instances on one bar, one
per niri desktop; `on-click` is waybar-level config (module registers no click
handler, only enter/leave, and propagates events).

| Severity | Failure | Why |
|---|---|---|
| S1 | Bar crash | in-process .so; kills every bar on the desktop. Both known crashes are S1 |
| S2 | Silently stale/wrong state | user misses a waiting prompt; three independent layers fail silent-stale; no staleness indicator exists anywhere |
| S3 | Attention mechanism failure (prompt shown but pulse/blink frozen) | `content_animates` heuristic; regressed twice for real |
| S4 | Visible error text (template error card) | ugly but honest |
| S5 | Cosmetic (hover, marquee, alignment) + heat regressions | felt as fan noise, not wrong data |

The attention-critical sub-state is `prompt`; the producer's prompt-first cap
and the whole-tile pulse are deliberate product investments (bead cek).

## 10. Unproven Assumptions Inventory `[assume]`

- "Engine is never dropped" (no guard; false — p9c).
- "waybar modules are process-lifetime" (stated twice; false under reload).
- "Stream stdout is valid UTF-8, one JSON doc per line, bounded, line-buffered"
  (each individually violable; §7 consequences).
- "GL object creation never fails" (shader.rs unwraps).
- "libepoxy.so.0 is loadable" (expect).
- "No GL ⇒ no draw at all is acceptable" (blanks CPU-only text).
- "A bad config degrades gracefully" — it does, to the *wrong tile* (60fps demo).
- "Icon load failure is permanent" (negative cache).
- f32 animation clock stays precise (ulp ~16ms @36h, ~125ms @12d, ~1s @97d
  uptime; ticker/pulse/blink visibly stutter then freeze; no rebase).
- `level='6'` literal in `content_animates` hardcodes agentic-db's
  `DecayLevels-1` — cross-repo constant coupling; an out-of-range
  `idle_level: 7` payload renders clamped but animates at 30fps forever (the
  heat bug reintroduced via data).
- Guards that DO exist: allocation/scale clamps, frame-clock overflow
  sentinel, wrapping frame counter, femtovg blend-state neutralization,
  monotonic Instant clocks (suspend-safe), atomic cache writes.

## 11. Wildcard / Cross-Cutting `[wild]`

- **F2 — in-band signaling**: U+FFFC in any data string injects a phantom
  embed and shifts embed indices (markup.rs:224-234 vs lib.rs:651-671 —
  "N placeholders ↔ N embeds" is load-bearing and unenforced); C0 control
  chars (invalid XML, not HTML-escaped away) in a window title collapse the
  whole tile to escaped tag soup; `\n` in data adds flow lines and shifts
  geometry. Window titles are app/webpage-influenced input.
- **F3 — two animation detectors that must agree** (`could_anim` on the
  template at init vs `content_animates` on rendered markup per update) are
  only coincidentally consistent; markup-borne animated tags (not literally in
  the template) would never get a tick callback.
- **F7 — three caches, zero invalidation protocol** across two processes:
  backend `iconMemo` (forever) → cache-dir SVG shim → plugin negative cache
  (forever). Delete `~/.cache` and the icon never returns until both processes
  restart.
- **F8 — data-controlled blocking read on the main thread**: `<icon src>` does
  synchronous `fs::read` in the draw callback (src/lib.rs:1248-1251); a FIFO
  path wedges the entire bar permanently; no size cap; arbitrary-file-read
  primitive for whoever controls the stream.
- **F9 — the alert function fails silent** (see §7 backend row) — the
  product's core purpose is the thing failures mask.
- **F10 — output identity**: cache keyed by connector name; renames on
  dock/lid events strand every `tile-watch` on a stale key; looks like "all
  desktops idle".
- **F11 — focus_bubble radius**: Cairo clamps `r` to half-extents, the GLSL
  mask doesn't — large `corner_radius` makes card and mask visibly disagree,
  falsifying a README "always agree" claim.
- **F13 — GL context etiquette**: the plugin never restores GDK's context; a
  second GL-using CFFI module would fight over thread-current context.
- **F14 minors**: `iFrame` frozen at 0 for `<bg>`/`<glow>` shaders; fixed
  `.tmp` name in backend cache write assumes exactly one daemon; doc drift
  (bar height, DEFAULT_ANIM_FPS 30 vs bead's 20).

## 12. Ranked Attack Surfaces for Antithesis

1. **Module teardown / GL context lifecycle** (S1, confirmed crash at this
   commit; trigger diversity: output removal, reload, exit).
2. **Host abort via draw-path panics** (S1: GL unwraps, epoxy expect, no
   catch_unwind anywhere).
3. **Reload accumulation** (threads/children/timers leak per reload; becomes
   the active reload bug once p9c fix merges).
4. **Silent staleness of the data chain** (S2: daemon death, cache deletion,
   hung producer, UTF-8 stall — all render as normality; prompt masked).
5. **Stream seam robustness** (torn lines, garbage, huge lines, kill/respawn
   timing, backoff behavior).
6. **Animation gating in both directions** (frozen attention pulse vs
   heat/CPU runaway; proven regression cluster; data-driven).
7. **Config/preset degradation** (type collapse → 60fps demo tile; typo →
   raw-JSON tile).
8. **Resource boundedness over long runs** (caches, GL object leaks, f32
   clock, fork storms).
9. **Contract conformance** (schema enforced nowhere; extreme-but-valid and
   invalid payloads both live inputs).

## Assumptions (of this analysis)

- Waybar behavior validated against upstream master + 0.14.0/0.15.0 tags for
  the CFFI paths; local binary is v0.15.0. Reload orchestration beyond the
  CFFI destructor was not traced verbatim.
- Rust unwind-out-of-`extern "C"` = abort asserted from language semantics

## Errata (post-investigation, 2026-07-22)

The open-questions investigation pass (6 agents; logs in
`properties/*.md` Investigation Logs) corrected or settled several claims
above:

- **§11 F2 / §7 (C0 control chars)**: the predicted Pango-unsafe fallback was

  (including NUL) with no warning and no abort. Severity settles at S2
  (whole-tile tag-soup collapse), not S1.
- **§1/§3 (`sh -c` exec-replace)**: dash does **not** exec-replace a single
  command — every producer chain is 2 processes (`sh` + `claude-status`).
  Count constants double; invariants unaffected. Host-specific; re-check the
  container's /bin/sh.
- **Open question "does SIGUSR2 reload destroy/re-init modules?" — resolved
  from waybar 0.15.0 source**: yes, unconditionally (no config diffing);
  destroy-all-then-construct-all (module generations never coexist); fresh
  dlopen per module; bars auto-recreated on output re-add (gated on config
  output match — a pinned `"output"` on a renamed connector yields *no bar*).
- **Open question "are `<glow>` shaders time-varying?" — resolved
  empirically**: yes (`sin(iTime*2.5)` alpha modulation, probe-verified), and

- **niri workspace indexes are renumbered** (not preserved) when an output is
